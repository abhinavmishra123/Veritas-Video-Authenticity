# Veritas Credential Format (v1)

## Overview
A Veritas Credential is a JSON document representing a cryptographically verified video capture session. It ensures that the video was captured on a trusted device and its frames remain unaltered.

## Schema

```json
{
  "version": 1,
  "video_id": "uuid-string",
  "device": {
    "public_key": "ed25519-public-key-hex",
    "type": "browser-webcrypto"
  },
  "capture": {
    "start_time": 1690000000000,
    "end_time": 1690000050000,
    "frame_count": 50
  },
  "frame_hashes": [
    {
      "sequence": 0,
      "timestamp": 1690000001000,
      "hash": "sha256-hex-string"
    }
  ],
  "signature": "ed25519-signature-hex"
}
```

## Signature Protocol
The `signature` field is generated by signing the JSON string representation of the credential (excluding the `signature` field itself) using the device's Ed25519 private key.

1. Serialize the JSON payload (excluding signature).
2. Encode as UTF-8 bytes.
3. Sign using Ed25519.
4. Convert signature to hex string.
